1. What is Paperclip?

Paperclip is an open-source AI agent orchestration platform built with Node.js and React. Its core idea is to run teams of AI agents as if they were employees of a company, working toward a shared mission.

Think of it as an operating system for AI agents. Instead of running a single chatbot, you define an entire organization — with roles, a reporting hierarchy, budgets, permissions, and a shared task queue — and let AI agents operate it autonomously, with human oversight baked in.

The company you're looking at in this installation is called "Catalyst" (note: CAT in the URL path is the company's identifier). It currently has one agent: a CEO running on Claude.

ℹ️ Quick Install You installed Paperclip by running npx paperclipai onboard --yes. The server runs on http://127.0.0.1:3100 and uses an embedded PostgreSQL database — no external setup needed.

2. Core Concepts & Mental Model

🏢
Company
The top-level container. Agents work for a company and share its goals, budget, and task board.
🤖
Agent
An AI entity with a role, instructions, and an adapter (e.g. Claude). Runs on heartbeats or when assigned a task.
📋
Issue
A unit of work. Can be created manually by humans or created by agents during a run. Agents check their inbox for assigned issues.
💓
Heartbeat
A scheduled, timed execution window for an agent. Every N seconds, the agent "wakes up", checks for work, does something, and exits.
🔧
Skill
A bundle of instructions or tools mounted into the agent's context at runtime. Skills teach agents how to use Paperclip's API.
🏃
Run
One execution of an agent. Has a full transcript, token counts, cost, invocation command, and environment variables.

The fundamental loop looks like this:

  1. You create a company with a mission.
  2. You add agents with roles and instructions.
  3. You or the agents create issues (tasks).
  4. Agents wake up on a heartbeat timer, find their assignments, and do the work.
  5. Each execution is a run — fully logged with token costs and transcript.
  6. When an agent needs human input, it creates an approval request.
💡 Key Insight Issues in Paperclip are not documentation — they are work orders for agents. An issue titled "Generate the CLAUDE.md for this project" is something an agent will literally read and execute. The agent's output and reasoning are visible right in the issue history.

4. Dashboard

The Dashboard is the home screen of a company. It refreshes in real-time and gives you a live window into what's happening.

Agents Feed (top section)

Shows the most recent run card for each agent in the company. Each card contains:

  • Agent name & initials badge — e.g. "CE" for CEO
  • Status indicator — gray circle = finished, blue circle = running now
  • Run timestamp — "Finished 43m ago" or "Live now"
  • Issue reference — short UUID of the issue that triggered the run (or blank for heartbeat runs)
  • Agent's reasoning text — the agent's last stdout output, shown as plain prose. During a live run, this streams in real-time.
  • STDOUT section — expandable technical output showing the raw JSON stream, model name, session ID, and any errors.
  • Expand tool details / Expand command details — collapsible panels for detailed execution info.
  • ↗ External link icon — navigates to the full run detail page.
💡 Reading Live Runs When the CEO runs on its heartbeat, you can watch it think in real time: "Good. I'm the CEO, company [id]. Heartbeat timer wake, no specific task. Let me check my inbox." When there's no work, it exits cleanly: "No assignments, no pending work, no mention handoffs."

6. Issues

Issues are the fundamental unit of work in Paperclip. They are both human-readable task definitions and the inputs that agents act on.

Issues List View

The toolbar at the top provides:

  • + New Issue — opens the issue creation modal
  • Search issues… — full-text search
  • List / Grid toggle — switch between a list and a board/grid view
  • Filter — filter by status, priority, assignee, label, project, etc.
  • Sort — sort by created date, updated date, priority, etc.
  • Group — group issues by status, priority, assignee, or project

Creating an Issue

Click + New Issue (sidebar or toolbar) or press C to open the creation modal:

The modal header shows the company prefix (CAT) — issues get sequential IDs like CAT-1, CAT-2, etc. You can expand the modal to full-screen using the ↗ icon, or discard it with Discard Draft.

💡 Writing Good Issues for Agents Write issue titles and descriptions the same way you would brief a human employee. Include context, expected output, and any constraints. Agents read the entire issue body before starting work.

7. Routines Beta

Routines are recurring work definitions that automatically generate issues on a schedule. The page describes them as: "Recurring work definitions that materialize into auditable execution issues."

Use routines when you have work that needs to happen repeatedly — for example:

  • A daily status summary from the CEO agent
  • A weekly report on costs and agent activity
  • A recurring code review cycle

Click + Create routine to define one. The routine will generate a real Issue each time it fires, giving you a full audit trail of every execution.

⚠️ Beta Feature Routines are tagged as Beta, meaning the feature is functional but the interface and behavior may change in future versions.

8. Goals

Goals are high-level company objectives. They sit above individual issues and give agents and humans a shared understanding of what the company is trying to achieve.

Click + Add Goal to create one. Goals can presumably be linked to issues and tracked for progress, though this relationship is not yet populated in this instance.

9. Activity

The Activity page (/CAT/activity) is a full chronological audit log of everything that has happened in the company. Every create, update, and delete event appears here.

Each row shows:

  • Actor badge — who performed the action. "BO" = Board (human operator), "CE" = CEO agent.
  • Action description — e.g. "Board updated Onboarding", "CEO commented on [issue]", "Board created"
  • Timestamp — relative time (e.g. "33m ago")

Use the All types dropdown (top right) to filter by event type.

ℹ️ "Board" Actor In Paperclip's event log, Board refers to you — the human operator. The "Board" terminology comes from the company metaphor: the human is the board of directors that oversees the AI agents. The "Board" creates and updates configuration, while agents (like "CEO") do the execution work.

10. Approvals

The Approvals page is where agents request human authorization before taking high-stakes actions. This is Paperclip's built-in human oversight mechanism.

Pending
All

When an agent determines that an action requires human approval (based on its instructions or permissions), it pauses and creates an approval request. The approval shows up here for the Board to review and either approve or reject.

The Pending Approvals KPI on the Dashboard links here and shows the current count. "Awaiting board review" appears under the counter when approvals exist.

13. Agent Instructions Tab

This is where you define who the agent is and what it knows. Instructions are markdown files that are injected into the agent's context at the start of every run.

Files Panel (left)

A list of all instruction files for this agent, with their sizes. The + button adds a new file. Each file shows:

  • File name — e.g. AGENTS.md, HEARTBEAT.md, SOUL.md, TOOLS.md
  • ENTRY badge — marks the primary entry-point file that gets loaded first (typically AGENTS.md)
  • File size — in bytes

File Viewer (right)

Clicking a file shows its markdown content. Non-ENTRY files have a Delete button. Files can be edited directly.

Default Files for a CEO Agent

FilePurpose
AGENTS.md ENTRY The main identity file. Tells the agent its role ("You are the CEO. Your job is to lead the company…"), where to store memory ($AGENT_HOME), and the delegation principle (CEO must delegate work to other agents, not do it themselves).
HEARTBEAT.md Instructions for what to do during a scheduled heartbeat wake. Defines the CEO's routine behavior when no specific task is assigned.
SOUL.md Personality, values, and behavioral guidelines for the agent.
TOOLS.md A growing list of tools available to the agent. Initially just a placeholder: "Your tools will go here. Add notes about them as you acquire and use them."
ℹ️ Agent Memory $AGENT_HOME is a special environment variable pointing to the agent's personal directory on disk. The agent can read and write files there across runs to maintain memory and context. Company-wide files (shared plans, docs) live in the project root instead.

Advanced Section

An expandable panel (> Advanced) with additional instruction configuration options. Contents are not fully explored in this guide.

14. Agent Skills Tab

Skills are bundles of instructions and tools that get mounted into the agent's temporary skill directory at the start of each run. They teach the agent how to use Paperclip's internal API.

The tab header includes a View company skills library link and shows the current adapter (Claude (local) in this case).

Required by Paperclip

These four skills are automatically included and cannot be removed:

paperclip-create-pluginGives the CEO the ability to create new Paperclip plugins.
para-memory-filesEnables the agent's memory file system for persistent knowledge across runs.

User-Installed Skills

An expandable section showing skills you've added yourself (in this instance: 4 user-installed skills). These can be toggled on/off per agent.

Skills Applied

At the bottom, a summary shows: Adapter (Claude (local)), Skills applied (when applied — "Applied when the agent runs"), and Selected skills count.

ℹ️ How Skills Work Skills are mounted into a temporary directory ($SKILL_DIR) at runtime. The SKILL.md inside each skill package is the instruction file the agent reads. Each skill entry in the list confirms: "Will be mounted into the ephemeral Claude skill directory on the next run."

15. Agent Configuration Tab

The Configuration tab is where you control the agent's identity, runtime behavior, permissions, and API access.

Identity

FieldDescription
NameThe agent's identifier (e.g. CEO). Used in URLs and references.
TitleA human-readable role title shown in the org chart (e.g. "VP of Engineering").
Reports toSets the agent's manager in the org hierarchy. Click to choose from other agents. A CEO agent has no manager.
CapabilitiesFree-text description of what this agent can do. Used to help other agents decide who to delegate tasks to.

Adapter

Specifies which AI model/runtime powers this agent. Currently Claude (local) is the adapter, meaning it runs the claude CLI tool installed on your machine. The Test environment button runs a quick environment check to verify the adapter is working correctly.

Additional adapter settings (visible when scrolled) include:

  • Max turns (default: 1000) — maximum number of tool use cycles per run
  • Extra args — comma-separated CLI arguments passed to the adapter
  • Environment variables — key/value pairs injected at runtime. Can be marked as Seal (encrypted/hidden in logs). Note: PAPERCLIP_* variables are injected automatically.
  • Timeout (sec) — hard time limit for a run (0 = no limit)
  • Interrupt grace period (sec) — how long after a pause request before the run is forcibly stopped (default: 15s)

Run Policy

16. Agent Runs Tab

The Runs tab is your complete execution history for the agent — every run, its outcome, and full technical details.

Run List (left panel)

Each run entry shows:

  • Status icon — ✅ green = succeeded, ❌ red = failed
  • Short run ID — first 8 chars of the UUID (e.g. 4f76af74)
  • Trigger type badge — On-demand, Assignment, Heartbeat
  • Relative timestamp — "51m ago"
  • Summary text — first line of the agent's output
  • Token count — e.g. "4.7k tok"

Run Detail (right panel)

Clicking a run shows:

SectionContents
Status + TimingResult (succeeded/failed), start time → end time, duration (e.g. 2m 1s).
Token countsInput / Output / Cached tokens and Total cost.
SessionExpandable — the Claude session ID for this run (used for session resumption).
InvocationThe exact command used to start the agent, including all CLI flags. Also shows the working directory ($PAPERCLIP_WORKSPACE_CWD).
EnvironmentAll environment variables passed to the agent. Key ones: PAPERCLIP_AGENT_ID, PAPERCLIP_API_KEY (redacted), PAPERCLIP_API_URL, PAPERCLIP_RUN_ID, PAPERCLIP_TASK_ID, PAPERCLIP_WAKE_REASON.
TranscriptThe full execution transcript. Toggle between Nice (formatted, readable) and Raw (JSON stream). Shows STDOUT with all model outputs, tool calls, and results.
ℹ️ PAPERCLIP_WAKE_REASON The environment variable PAPERCLIP_WAKE_REASON tells the agent why it was activated. Values include heartbeat (scheduled timer), assignment (a task was assigned), retry_failed_run (retrying a previous failed run), and on_demand (manually triggered).

18. Adding a New Agent

Click the + button next to "AGENTS" in the sidebar, or use the command palette (search → "Create new agent").

A dialog appears with two paths:

🤖 Ask the CEO to create a new agent

The recommended approach. The CEO already knows the org structure and can configure reporting, permissions, and adapters. This triggers the CEO agent immediately to create and configure the new agent on your behalf.

⚙️ I want advanced configuration myself

Takes you directly to the manual agent setup form where you fill in all configuration fields yourself (Name, Title, Reports To, Capabilities, Adapter, Run Policy, etc.).

💡 Letting the CEO Hire In the Paperclip philosophy, the CEO agent is responsible for building out the team. If you ask the CEO to create a new agent (e.g., by creating an issue: "Hire a Software Engineer agent"), the CEO will use its paperclip-create-agent skill to set up the new agent properly.

19. Org Chart

The Org Chart (/CAT/org) renders a visual hierarchy of all agents in the company.

Features:

  • Each agent appears as a node showing: icon, name, role, adapter (e.g. "Claude"), and a status dot (yellow = running/active)
  • Reporting lines are drawn as edges between nodes
  • Zoom controls (+, −, Fit) in the top-right corner
  • Import company — load a pre-built company template (from the paperclipai/companies GitHub repo)
  • Export company — export the current company config as a template to share or back up
ℹ️ Company Templates Paperclip maintains a public repository of pre-built company templates at github.com/paperclipai/companies. You can import these to instantly set up a fully configured multi-agent organization (e.g. a software startup with CEO, CTO, and developer agents).

What the Core Paperclip Skill Contains

The paperclip skill's SKILL.md teaches agents:

  • They run in heartbeats — short windows, not continuously
  • How to authenticate via auto-injected env vars (PAPERCLIP_AGENT_ID, PAPERCLIP_COMPANY_ID, PAPERCLIP_API_URL, PAPERCLIP_RUN_ID)
  • Optional wake-context vars: PAPERCLIP_TASK_ID (issue that triggered the wake) and others

23. Instance Settings — Heartbeats

The Heartbeats settings page gives you a global view of all scheduled agent heartbeats across every company on this Paperclip instance.

The page shows:

  • Summary counts: N active, N disabled, N companies
  • A Disable All button — emergency stop for all scheduled runs
  • Per-company sections listing each agent with: name, slug, interval, last run time, and a Disable Timer Heartbeat button

Example: Under "CATALYST", you'd see: On CEO · ceo · 3600s · 53m ago · [Disable Timer Heartbeat]

⚠️ Use "Disable All" with Care Disabling all heartbeats stops all automatic agent activity immediately. This is useful during maintenance but will prevent agents from picking up work until re-enabled.

26. Creating a New Company

Click the + button at the very bottom of the left icon strip to launch the new company setup wizard. This is a 4-step guided flow:

🏢 Company
🤖 Agent
📋 Task
🚀 Launch

Step 1 — Company

"Name your company — this is the organization your agents will work for."

  • Company name — e.g. "Acme Corp" (required)
  • Mission / goal (optional) — "What is this company trying to achieve?" This is injected into the agent's context so it understands the purpose of its work.

Step 2 — Agent

Configure the first (CEO) agent: adapter selection, model preferences, etc.

Step 3 — Task

Optionally provide an initial task/issue to give the agent when it first wakes up.

Step 4 — Launch

Review and confirm. Paperclip sets up the database, creates the company, configures the CEO, and starts the heartbeat scheduler.

28. Things That Are Unclear

During this exploration, several things could not be fully understood. Rather than guess, they are documented here so you can investigate further.

❓ Issue Detail Page Renders Blank Navigating to a specific issue by UUID (e.g. /CAT/issues/5e7cd541-...) resulted in a blank content area. It's unclear whether this is a known rendering bug, a data issue specific to this particular issue, or if issue detail pages require a different rendering condition (e.g. the issue must be in a specific state). The short-form identifier (/CAT/issues/CAT-1) also rendered blank.
❓ Projects Section The PROJECTS section in the sidebar has a + button, and the command palette has "Create new project." However, the project creation flow was not triggered successfully during this exploration (the button appeared to require a hover state). The relationship between Projects, Issues, and agent behavior is not fully documented here — it's likely that Projects are used to group Issues, but whether agents treat projects as prioritized units of work is unclear.
❓ Routines "Create Routine" Form Clicking "+ Create routine" on the Routines page did not open a modal or navigate away during testing. The feature is labeled Beta, so this may be a known limitation. The exact fields for defining a routine (schedule, assignee, template issue, etc.) were not observed.
❓ Costs — Finance Net and Finance Events The "Finance Net" and "Finance Events" cards on the Costs page display values that suggest a more formal billing system (debits, credits, invoice events). The specific scenarios that generate Finance Events (as opposed to Inference Spend events) are not clear. This may relate to paid Paperclip Cloud features not active in local installations.
❓ Agent Instructions — "Advanced" Section The Instructions tab has a collapsible > Advanced section that was not explored in depth. It likely contains settings for instruction ordering, context window management, or system prompt structure — but this is speculative.
❓ Configuration Revisions The agent Configuration tab shows "Configuration Revisions 2" as an expandable section. The exact format of revisions, whether you can roll back to a previous config, and how diffs are shown were not explored.
❓ Documentation "v" Indicator The Documentation link at the bottom of the sidebar shows a small "v" next to it. This likely indicates the installed Paperclip version, but the exact version string was not visible in the viewport during exploration.
